设置Git

# 设置 Git

GitHub 的核心是名为 Git 的开源版本控制系统 (VCS) 。 Git 负责在您计算机上本地发生的、与 GitHub 有关的所有内容。

# 使用 Git

要在命令行中使用 Git，您将需要在计算机上下载、安装和配置 Git。 您还可以安装 GitHub CLI 以从命令行使用 GitHub。 更多信息请参阅“关于 GitHub CLI (opens new window)”。

如果要在本地使用 Git，但不想使用命令行，您可以下载并安装 GitHub Desktop (opens new window) 客户端。 更多信息请参阅“安装和配置 GitHub Desktop (opens new window)”。

如果无需在本地使用文件，GitHub 可让您在浏览器中直接完成许多 Git 相关的操作，包括：

  • 创建仓库 (opens new window)
  • 复刻仓库 (opens new window)
  • 管理文件 (opens new window)
  • 社交化 (opens new window)

# 设置 Git

  1. 下载并安装最新版本的 Git (opens new window)。

    注意：如果您使用的是 Chrome OS 设备，则需要进行额外设置：

    1. 在你的Chrome OS设备上安装一个终端模拟器，比如谷歌Play商店的Termux
    2. 从您安装的终端模拟器中，安装Git。例如，在Termux中，输入apt install git，然后在提示时键入y。

  2. 在 Git 中设置您的用户名 (opens new window)。

  3. 在 Git 中设置提交电子邮件地址 (opens new window)。

# 后续步骤：使用来自 Git 的 GitHub 进行身份验证

从 Git 连接到 GitHub 仓库时，您将需要使用 HTTPS 或 SSH 通过 GitHub 进行身份验证。

注意：您可以使用GitHub CLI对GitHub进行HTTP或SSH身份验证。有关更多信息，请参阅gh auth login (opens new window)。

# 通过 HTTPS 连接（推荐）

如果使用 HTTPS 克隆 (opens new window)，您可以使用凭据小助手在 Git 中缓存 GitHub 凭据 (opens new window)。

# 通过 SSH 连接

如果使用 SSH 克隆 (opens new window)，您必须在用于从 GitHub 推送或拉取的每台计算机上生成 SSH 密钥 (opens new window)。

# 祝贺

恭喜。您现在已将 Git 和 GitHub 全部设置完毕！ 您现在可以选择创建仓库以放置项目。 这是备份代码的好方法，易于在世界各地分享代码。 更多信息请参阅“创建仓库 (opens new window)”。

您可以通过复刻创建仓库的副本，并提出您希望看到的更改，而不会影响上游仓库。 更多信息请参阅“复刻仓库 (opens new window)”。

GitHub上的每个存储库都由一个人或一个组织拥有。 您可以在 GitHub 上连接和关注人员、仓库和组织以与之进行交互。 更多信息请参阅“社交 (opens new window)”。

GitHub 有一个强大的支持社区，您可以在那里寻求帮助并与来自世界各地的人交谈。 加入 Github 支持社区 (opens new window)的对话。
